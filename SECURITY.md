Reporting a Vulnerability
If you discover a security vulnerability within uschecker, please report it responsibly. Do not disclose vulnerabilities publicly in a way that exposes users before a fix or patch has been provided.

How to Report
Private Reporting: Please contact the repository maintainer privately or open a private security advisory on GitHub rather than creating a public issue.

Information to Include:

A clear description of the vulnerability and its potential impact.

Step-by-step instructions or a proof-of-concept (PoC) to reproduce the issue.

Any suggestions or ideas for a potential fix or mitigation.

What to Expect
Acknowledgment: You will receive an initial response acknowledging your report within 48 to 72 hours.

Updates: You can expect periodic updates as the issue is evaluated and fixed.

Resolution: Once the vulnerability is confirmed and addressed, a patched version or security advisory will be released, and credit will be given to you as the reporter (unless you prefer to remain anonymous).
